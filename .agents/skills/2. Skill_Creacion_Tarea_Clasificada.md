# Skill: Creación de Tareas Clasificadas

Este skill permite procesar requerimientos y realizarlos en Historias de Usuario.

## Flujo de Trabajo

1. dentro de la carpeta data procesada, con estado en historia de usuario "Para procesar"

2. **Procesamiento:** El agente analiza la solicitud, redacta la Historia de Usuario y desglosa las actividades.
3. **Salida:** El archivo Markdown resultante se debe guardar en la carpeta Historia de Usuarios.
4. **Actualizacion de estado:** dentro de la carpeta data procesada, con estado en historia de usuario "Procesado"

## Contexto de Clasificación (Matriz de Referencia)

Para aplicar este skill, el requerimiento debe encajar obligatoriamente en uno de los siguientes subtipos de la clasificación **Tarea**:

| Clasificación | Tipo de Requerimiento | Definición | Prioridad Sugerida |
| :--- | :--- | :--- | :--- |
| **🛠️ Tarea** | Operativo / Técnico | Trabajo práctico inmediato o resolución de un problema técnico puntual (ej. configurar un acceso, corregir un error menor). | **Alta** |
| **🛠️ Tarea** | Administrativo | Actividades de gestión de documentos, firmas o aprobaciones inmediatas. | **Media** |

## Instrucciones para el Agente

Cuando proceses una solicitud para crear una "Tarea Clasificada", debes realizar los siguientes pasos de análisis y estructuración:

1. **Validación del Tipo:** Clasifica el requerimiento como *Operativo / Técnico* o *Administrativo*.
2. **Definición del Título:** Asigna un nombre claro, conciso y de acción directa.
3. **Redacción de la Historia de Usuario Básica:** Traduce la necesidad operativa a una Historia de Usuario en formato estándar:
   - **Como:** [Rol del usuario final o solicitante]
   - **Quiero:** [Acción concreta o capacidad técnica que se requiere]
   - **Para:** [El beneficio de negocio, valor generado o problema que se resuelve]
4. **Establecimiento de Criterios de Aceptación:** Define al menos 2-3 criterios medibles y claros que determinen cuándo la tarea está completada exitosamente.
5. **Plan de Acción / Desglose de Pasos:** Describe las sub-tareas técnicas o pasos administrativos necesarios para ejecutar la tarea.
6. **Prioridad y Asignación:** Define la prioridad sugerida (Alta o Media según la matriz) y determina el responsable (si se menciona en la fuente).
